# Roadmap - План развития

## Бизнес-модель

### Целевая аудитория
Косметологи, массажисты, стоматологи, репетиторы, тренеры и другие специалисты, работающие по записи

### Тарифная сетка

| Тариф | Цена | Условие |
|-------|------|---------|
| **Индивидуальный** | 300 грн/мес | 10+ записей в месяц |
| **Компания (3-9 специалистов)** | 250 грн/специалист | от 3 специалистов |
| **Компания (10+ специалистов)** | 200 грн/специалист | от 10 специалистов |

*Бесплатный период для новых пользователей — 14 дней*

## Критический приоритет (архитектурное изменение)

### 0. Клиентский портал /client + Мульти-специалист

**Проблема:** Один клиент может ходить к косметологу, стоматологу и тренеру. Сейчас клиент привязан к одной компании.

**Решение:**

#### База данных
- [ ] Создать таблицу `client_companies` (many-to-many связь)
- [ ] Миграция данных из `clients.company_id`
- [ ] Убрать `company_id` из `clients` (или оставить как "основную")

#### Client Bot
- [ ] При переходе по новой invite_link — добавлять связь, а не заменять
- [ ] Показывать выбор специалиста если их несколько
- [ ] "Мои записи" — группировать по специалистам

#### Клиентский портал (frontend /client)
- [ ] Авторизация через Telegram Widget
- [ ] Список всех специалистов клиента
- [ ] История записей у каждого
- [ ] Возможность записаться к любому
- [ ] Отмена записи

### 0.1 Superadmin панель /superadmin

**Цель:** Панель для владельца платформы (тебя) для управления и монетизации.

#### База данных
- [ ] Таблица `subscriptions` (подписки компаний)
- [ ] Таблица `payments` (история платежей)
- [ ] Поле `is_superadmin` в `users` или отдельная таблица

#### Superadmin frontend
- [ ] Авторизация (отдельная или по email/telegram)
- [ ] Дашборд со статистикой платформы:
  - Всего компаний / специалистов / клиентов
  - Новые регистрации за период
  - Активные пользователи (по записям)
- [ ] Список компаний:
  - Название, тип, дата регистрации
  - Количество специалистов / клиентов / записей
  - Статус подписки (активна/истекла/trial)
  - Сумма оплат
- [ ] Управление подписками:
  - Активация/деактивация
  - Ручное продление
- [ ] История платежей по компаниям
- [ ] Экспорт данных (CSV/Excel)

#### Backend API
- [ ] Эндпоинты для superadmin статистики
- [ ] Middleware проверки superadmin прав
- [ ] Интеграция с платежами (Monobank/LiqPay)
